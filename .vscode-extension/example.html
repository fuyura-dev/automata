Transformation Rules Syntax Highlighting Example

Syntax Elements:

// Comments - Green, italic
C V R - Blue, bold (Variables: Consonant, Vowel, Root)
"affix" - Orange (Quoted affixes)
-> - Purple, bold (Arrow operator)
. - White (Separator)
{"key": "value"} - Light blue (JSON metadata)
// ******************** -Um- verbs C "um" V R -> C V R . {"form": ["infinitive", "completed"] } // tumakbo C "um" V C V R -> C V R . {"form": "incompleted" } // tumatakbo C V C V R -> C V R . {"form" : "contemplated"} // tatakbo "um" V R -> V R . {"form": ["infinitive", "completed"] } // uminom // ******************** Ma- verbs "ma" R -> R . {"form": "infinitive"} // manood "na" R -> R . {"form": "completed"} // nanood // ******************** -In verbs R C "in" -> R C . {"form": "infinitive"} // kainin C "in" R -> C R . {"form": "completed"} // kinain

How the Extension Works

The VSCode extension provides syntax highlighting for .rules files by:

  • Recognizing variables (C, V, R) and highlighting them
  • Highlighting quoted affixes like "um", "mag", etc.
  • Highlighting the arrow operator ->
  • Providing JSON syntax highlighting for metadata sections
  • Highlighting line comments starting with //
